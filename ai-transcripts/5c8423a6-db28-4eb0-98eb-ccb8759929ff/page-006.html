Thinking Timeline + Clean Output Separation

Problem

The agent's intermediate reasoning ("Good — solid foundation. Now let me run targeted sentiment...") gets concatenated into the same string as the final analysis output. The user sees a stream-of-consciousness blob instead of a clean document. Meanwhile, tool calls sit in a separate accordion disconnected from the thinking that triggered them.

Goal

Separate the agent's work into two distinct layers:

  1. Timeline — an ordered sequence of thinking steps and tool calls, rendered as a collapsible process log above the output. This is where the user sees how the agent worked: reasoning interleaved with tool activity, in chronological order.

  2. Output — the final clean analysis, rendered as the main message content with inline charts, citations, and follow-ups. No process narration.

The rendered message structure becomes:

[Collapsible timeline: thinking + tool calls interleaved chronologically]
[Clean output with inline charts and citations]
[Follow-up buttons]

Replace tool_calls with timeline: list[TimelineStep] on ChatMessage. Keep backward compat — the tool_calls field can stay as a deprecated alias or just remove it since nothing persists long-term.

app/agent.py

This is the core change. Instead of accumulating all TextBlocks into one response_text string:

  • Maintain a timeline: list[dict] and a chunks: list[str] during the agent run
  • Each time an AssistantMessage arrives with a TextBlock, append the text to a current thinking buffer
  • Each time a tool fires (detected via the _emit_tool SSE events — use tool_records as the signal), flush the current thinking buffer as a {"type": "thinking", "text": "..."} entry in the timeline, then add the tool record as {"type": "tool", ...}
  • After the agent run completes, the last text chunk (after all tools have fired) is the output content. Everything before it was thinking.

Concrete approach: since tool_records is populated by the tool closures during execution, and TextBlocks arrive via the message stream, track the count of tool_records after each TextBlock. If new tool records appeared since the last TextBlock, that means the previous text was thinking (it preceded tool calls). The final TextBlock that has no subsequent tool calls is the output.

Simpler approach that might work better: collect ALL text chunks in order. After the run completes, take the last chunk as content and all prior chunks as thinking entries. Interleave thinking entries with tool_records based on their position — thinking[0] happened before tool_records[0:N], thinking[1] happened before tool_records[N:M], etc.

The simplest correct approach: maintain a single ordered steps list. Every TextBlock appends a thinking step. Every tool_record (already tracked via the accumulator) appends a tool step. After the run, pop the last thinking step off and use it as content. The rest of steps is the timeline.

To make the interleaving work, the tool emissions need to add to the same ordered list. Currently tool_records is a flat list filled by closures. Instead, pass the timeline list to create_review_tools_server and have _emit_tool append directly to it. TextBlocks in the agent loop also append to it. Order is preserved because everything runs sequentially in the async loop.

# How many recent messages to pass in full before summarising

# Max older messages to summarise
SUMMARY_WINDOW = 30


def _build_conversation_context(
    conversation_history: list[ChatMessage],
    user_message: str,
) -> str:
    """Build structured conversation context for the agent.

    Strategy:
    - Recent messages (last RECENT_WINDOW) are passed in full with role labels
    - Older messages are compressed into a topic summary
    - A session context block tracks what's been explored
    """
    total = len(conversation_history)
    parts: list[str] = []

    if total > 0:
        # Split into older and recent
        recent_start = max(0, total - RECENT_WINDOW)
        older = conversation_history[max(0, recent_start - SUMMARY_WINDOW):recent_start]
        recent = conversation_history[recent_start:]

        # Summarise older messages as topic bullets
        if older:
            topics = _extract_topics(older)
            parts.append(
                "## Session Context\n"
                f"This is message {total + 1} in the conversation. "
                f"Earlier topics explored:\n{topics}"
            )

        # Pass recent messages with structure
        if recent:
            lines = []
            for msg in recent:
                role = "User" if msg.role == "user" else "Assistant"
                # Truncate very long assistant responses in context
                content = msg.content
                if msg.role == "assistant" and len(content) > 800:
                    content = content[:800] + "\n[... truncated for context ...]"
                lines.append(f"**{role}:** {content}")
            parts.append("## Recent Conversation\n" + "\n\n".join(lines))

    parts.append(f"## Current Question\n{user_message}")
    return "\n\n".join(parts)


def _extract_topics(messages: list[ChatMessage]) -> str:
    """Extract topic bullets from older messages for context summary."""
    topics: list[str] = []
    for msg in messages:
        if msg.role == "user":
            # Use first 120 chars of user messages as topic indicators
            text = msg.content.strip()
            if len(text) > 120:
                text = text[:120] + "..."
            topics.append(f"- {text}")
    if not topics:
        return "- (general exploration)"
    return "\n".join(topics)

        name="broaden_search",
        description="Takes a topic and generates 4-5 query variants (synonyms, phrasings, related terms) via Haiku, batch-executes them against the review database, deduplicates, and returns merged results with a coverage summary.             "type": "object",
            "properties": {

                    "type": "string",
                    "description": "The topic to search broadly — e.g. 'shipping speed', 'customer service quality', 'food freshness'.",

                "max_rating": {"type": "number", "description": "Optional rating filter."},
            },
            "required": ["topic"],
        },

    async def broaden_search_tool(args: dict[str, Any]) -> dict[str, Any]:
        topic = args["topic"]
        n_per = min(args.get("n_per_query", 8), 15)

        # Use Haiku to generate query variants
        client = anthropic.AsyncAnthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))
        expansion_resp = await client.messages.create(
            model=HAIKU_MODEL,
            max_tokens=200,
            messages=[{
                "role": "user",
                "content": (
                    f"Generate 5 search query variants for finding customer reviews about: \"{topic}\"\n"
                    "Include synonyms, related phrases, and different angles someone might use to discuss this topic.\n"
                    "Return ONLY a JSON array of strings, nothing else. Example: [\"query 1\", \"query 2\", ...]"
                ),
            }],
        )
        raw = expansion_resp.content[0].text.strip()

        # Parse the query variants
        try:
            queries = json.loads(raw)
            if not isinstance(queries, list):
                queries = [topic]
        except json.JSONDecodeError:
            queries = [topic]

        # Always include the original topic
        if topic not in queries:
            queries.insert(0, topic)
        queries = queries[:5]

        # Build where clause for rating filters
        conditions = []
        if "min_rating" in args:
            conditions.append({"rating": {"$gte": args["min_rating"]}})
        if "max_rating" in args:
            conditions.append({"rating": {"$lte": args["max_rating"]}})
        where = None
        if len(conditions) > 1:
            where = {"$and": conditions}
        elif len(conditions) == 1:
            where = conditions[0]

        # Execute all queries and deduplicate

        merged: list[dict] = []
        query_hits: dict[str, int] = {}

            results = vectordb.search_reviews(session_id, q, n_results=n_per, where=where)
            new_count = 0
            for r in results:
                if r["id"] not in seen_ids:
                    seen_ids.add(r["id"])
                    merged.append(r)
                    new_count += 1
            query_hits[q] = len(results)
