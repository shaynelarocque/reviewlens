Based on my review of the official Claude Code documentation, I have to be honest with you: **there is no built-in "extract transcripts" command or feature in Claude Code**. There's no `claude-code-transcript`, transcript export tool, or similar command.

Here's what you should know:

**What IS available for preserving conversations:**
1. **Session persistence** - Claude Code automatically saves your sessions and lets you resume them with `claude -c` or `claude --resume <name>`
2. **Session naming** - Name your sessions with `claude -n "my-session-name"` so they're easier to find later
3. **Checkpointing** - Use `Esc+Esc` to rewind or summarize conversations at any point
4. **CLAUDE.md and auto memory** - These capture learnings across sessions, but they're not full transcripts

**Why transcripts aren't exported:**
- Claude Code is designed as an interactive CLI experience, not for transcript generation
- Conversations are stored locally in session data, not as standalone transcript files
- The focus is on continuation and context management rather than export

**Workarounds:**
1. **Terminal recording** - Use terminal recording tools like `script` (Unix/macOS):
   ```bash
   script -r output.txt claude
   ```
   This captures everything you and Claude type/output.

2. **Copy from terminal** - Manually select and copy conversation history from your terminal window

3. **Custom hooks** - If you really need to capture transcripts, you could potentially write a hook that logs conversation turns to a file, though this isn't documented as a standard feature

4. **Resumable sessions** - Since Claude Code saves sessions fully, you can always resume and reference past work without needing an explicit transcript export

If you need transcript export as a feature, I'd recommend filing a feature request using `/feedback` in a Claude Code session. This would help Anthropic understand the use case and prioritize it for future development.

# Claude Code Transcripts: Installation and Usage

## Installation

Install via `uv`:

```bash
uv tool install claude-code-transcripts
```

Or run without installing:

```bash
uvx claude-code-transcripts --help
```

## Core Commands

The tool provides four main commands for converting Claude Code sessions to HTML transcripts:

1. **`local`** (default) - Select from sessions stored in `~/.claude/projects`
2. **`web`** - Import sessions via the Claude API
3. **`json`** - Convert specific JSON or JSONL files
4. **`all`** - Convert all local sessions into a browsable archive

## Quick Start

Run `claude-code-transcripts` with no arguments to interactively select a recent local session. The tool generates HTML and opens it in your browser automatically.

## Output Options

Common flags across all commands:
- `-o, --output DIRECTORY` - Specify output location
- `-a, --output-auto` - Auto-name subdirectories by session ID
- `--open` - Open generated HTML in browser
- `--gist` - Upload to GitHub Gist for sharing
- `--json` - Include the original session file

## Key Features

The tool produces "clean, mobile-friendly HTML pages with pagination," generating an index page plus numbered transcript pages (page-001.html, page-002.html, etc.).

**Note:** The `web` command for importing sessions is currently broken due to API changes.
